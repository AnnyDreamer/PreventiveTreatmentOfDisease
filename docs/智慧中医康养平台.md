> **版本**：v1.0
> **更新日期**：2026-02-26

## 一、产品概述

### 1.1 产品愿景

> "上工治未病，中工治欲病，下工治已病。" —— 《黄帝内经》

**智慧中医康养平台**是一款融合中医体质辨识智慧与现代 AI 技术的全链路中医康养管理平台，面向医院治未病科/慢病管理科，提供从体质评估、个性化养生方案、日常健康管理到智能随访的完整闭环解决方案。

### 1.2 核心问题

| 问题维度 | 现状痛点 |
|---------|---------|
| **慢病困局** | 中国慢性病患者超4亿，占总死亡人数88.5%，但慢病管理依从性不足30% |
| **医院痛点** | 患者"看完就走"，院外管理完全断裂，医生无法追踪治疗效果，康养科创收能力弱 |
| **患者痛点** | 离院后无人管理，养生建议笼统（"清淡饮食"），缺乏可执行的具体方案 |
| **政策要求** | 国家卫健委《AI应用场景参考指引》明确"AI+中医药"6大场景为重点方向 |

### 1.3 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| **患者（C端）** | 慢病/亚健康人群，关注中医养生 | 了解自身体质，获得可执行的养生方案，有人持续关注健康 |
| **医生（B端）** | 治未病科/中医科/康养管理科医生 | 高效管理患者群体，追踪治疗效果，降低随访工作量 |
| **医院管理者（B端）** | 院长/科室主任 | 提升复诊率和科室创收，响应政策数字化要求 |

### 1.4 商业模式（B2B2C）

```
医院买单 → 患者使用 → AI提升粘性 → 提高复诊率 → 医院增收（正向飞轮）
```

- **收费方**：医院（SaaS订阅 / 定制化部署）
- **使用方**：患者（微信小程序免费使用）+ 医生（Web管理后台）
- **价值链**：医院采购 → 部署系统 → 医生引导患者使用 → 患者持续活跃 → 复诊率提升 → 医院增收

### 3.2 患者端功能（微信小程序）

#### F1：中医体质评估

**功能描述**：基于《中医体质分类与判定》国家标准的体质评估系统。

| 子功能 | 说明 |
|--------|------|
| 标准化问卷 | 9种体质 × 67道题目，5级评分（没有/很少/有时/经常/总是） |
| 智能评分 | 转化分公式：`(原始分 - 条目数) / (条目数 × 4) × 100` |
| 体质判定 | 平和质 ≥ 60 且偏颇 < 30 为平和；偏颇 ≥ 30 为该体质 |
| AI增强分析 | 通义千问分析体质特征、健康风险、兼夹体质交互影响 |
| 可视化报告 | 9体质得分柱状图 + 主/兼夹体质标签 + 特征描述 + 易患疾病 |
| 历史记录 | 多次评估对比，追踪体质变化趋势 |

**9种中医体质类型**：

| 体质 | 代码 | 颜色标识 | 特征简述 |
|------|------|---------|---------|
| 平和质 | BALANCED | 🟢 #22c55e | 身体健壮，精力充沛，适应能力强 |
| 气虚质 | QI_DEFICIENCY | 🟡 #f59e0b | 易疲劳，气短懒言，免疫力较低 |
| 阳虚质 | YANG_DEFICIENCY | 🔵 #6366f1 | 畏寒怕冷，手足不温，精神不振 |
| 阴虚质 | YIN_DEFICIENCY | 🔴 #ef4444 | 口燥咽干，手足心热，面色潮红 |
| 痰湿质 | PHLEGM_DAMPNESS | 🟣 #a855f7 | 体形肥胖，腹部肥满，口黏苔腻 |
| 湿热质 | DAMP_HEAT | 🟠 #f97316 | 面垢油光，口苦口干，身重困倦 |
| 血瘀质 | BLOOD_STASIS | 💜 #7c3aed | 肤色晦暗，色素沉着，易生瘀斑 |
| 气郁质 | QI_STAGNATION | ⚪ #64748b | 情绪低落，多愁善感，胸胁胀满 |
| 特禀质 | SPECIAL | 🩷 #ec4899 | 先天禀赋异常，易过敏 |

**双轨制评估流程**：

```
问卷填写(67题) → 国标算法确定性评分 → AI增强智能分析 → 可视化报告
                  ↓                      ↓
             体质类型+分数          特征解读+风险提示+调理方向
```

#### F2：个性化养生方案

**功能描述**：AI基于体质评估结果自动生成五维个性化养生方案。

| 维度 | 内容示例（气虚质） |
|------|------------------|
| 🥘 饮食调养 | 推荐：山药粥、黄芪鸡汤；避免：生冷寒凉、油腻；代茶饮：黄芪+枸杞+红枣 |
| 🏃 运动指导 | 八段锦、太极拳、散步；每次30分钟、每周5次；避免剧烈运动和大汗 |
| 🌙 起居调摄 | 早睡早起、23点前入睡；注意保暖、避免受风；午休15-30分钟 |
| 🧘 情志养护 | 保持乐观心态；避免思虑过度；可听古琴、做冥想 |
| 📍 穴位保健 | 足三里（补气健脾）、气海（益气培元）；每穴按揉3-5分钟 |

**方案特点**：
- AI基于中医知识库 + 个人体质类型 + 年龄/性别/既往史自动生成
- 同样体质类型，不同年龄/职业获得的方案细节完全不同
- Zod Schema + generateObject 确保AI输出100%结构化
- 方案随体质变化动态更新

#### F3：健康日记

**功能描述**：四维健康打卡记录系统。

| 记录维度 | 说明 |
|---------|------|
| 🛏️ 睡眠记录 | 时长（小时）+ 质量评分（1-5星），追踪入睡时间变化 |
| 😊 情绪打卡 | 5级emoji评分，发现情绪波动规律 |
| 🏃 运动记录 | 时长 + 类型（散步/太极/八段锦…），对比养生方案建议 |
| 💊 症状记录 | 16种常见症状标签多选 + 文字备注 |

**附加功能**：
- 日历热力图可视化连续打卡
- 打卡数据自动同步给主治医生
- 趋势图表展示改善情况

#### F4：AI 中医健康助手

**功能描述**：7×24小时的个性化中医养生顾问。

**核心能力**：
- **深度上下文注入**：每次对话自动注入用户体质类型、近期日记数据、当前养生方案
- **体质感知回答**：所有建议都基于用户具体体质，而非通用回答
- **日记数据关联**：结合近期健康数据给出个性化建议
- **紧急症状拦截**：30+ 紧急关键词硬编码拦截（胸痛、中风、大出血等），立即引导就医

**对话示例**：
```
用户：黄芪泡水每次放多少克合适？我是气虚体质。
AI：根据您的气虚质体质，建议黄芪泡水每次用10-15克，搭配3-5颗红枣和
    几粒枸杞效果更佳。⏰ 建议上午饮用，一周喝5天休2天。
    ⚠️ 如果您有感冒发热症状，暂停饮用。

用户：最近总觉得提不起劲
AI：结合您近一周的日记记录，睡眠质量波动较大(2-4分)，这可能是乏力的
    主要原因。建议：
    1. 今晚试试22:30前入睡
    2. 午后做10分钟八段锦
    3. 晚餐加一碗山药莲子粥
```

**安全边界**：
- 只提供养生建议，不做疾病诊断
- 不开具处方，不替代医生专业诊疗
- 超出能力范围诚实告知并建议就医
- 规则引擎优先级高于AI模型输出

#### D5：风险预警中心

**功能描述**：AI自动识别风险信号，医生精准介入。

**风险信号来源**：

| 来源 | 触发条件 |
|------|---------|
| 📔 日记异常 | 连续3天睡眠 < 5h；情绪评分连续下降；新增症状 |
| 🔄 随访预警 | AI分析反馈评分低于3分；新出现症状；依从性骤降 |
| ⏰ 行为预警 | 超过14天未打卡；超过30天未复诊；错过随访 |
| 🤖 对话预警 | AI对话中提及胸痛/头晕等敏感症状（已引导就医并上报） |

**医生干预流程**：
1. 看板醒目提示 → "12名高风险患者"红色角标
2. AI摘要辅助判断 → 自动生成患者风险概述
3. 一键触达患者 → 推送随访提醒/复诊建议到小程序
4. 闭环追踪 → 复诊后重新评估，风险等级动态调整

#### D6：统计报表

| 报表 | 内容 |
|------|------|
| 体质分布统计 | 各体质类型的人数及占比 |
| 复诊率分析 | 月度/季度复诊率趋势及对比 |
| 随访完成率 | 各计划的随访完成情况统计 |

### 3.4 AI 智能中枢

#### A1：体质辨识引擎

- **输入**：67道问卷答案 + 用户基本信息
- **处理**：国标算法评分 → 通义千问增强分析
- **输出**：主体质类型、兼夹体质、分数明细、特征解读、风险提示
- **技术**：Zod Schema + generateObject 确保结构化输出

#### A2：方案生成引擎

- **输入**：体质评估结果 + 用户画像（年龄/性别/既往史）
- **处理**：基于中医知识库的prompt工程 + 结构化输出
- **输出**：五维养生方案（饮食/运动/起居/情志/穴位）
- **特点**：同体质不同人群获得差异化方案

#### A3：健康问答引擎

- **上下文注入**：体质类型 + 近期日记 + 养生方案 + 对话历史
- **能力边界**：养生建议 ✓ | 疾病诊断 ✗ | 处方开具 ✗
- **安全机制**：30+ 紧急关键词硬编码拦截，规则引擎优先级 > AI输出

#### A4：随访分析引擎

- **输入**：患者随访反馈（评分/症状/自述）
- **处理**：AI分析反馈内容，与历史数据对比
- **输出**：风险标记（riskFlag）+ 风险说明 + 建议摘要

#### A5：风险预警引擎

- **多源数据融合**：日记数据 + 随访反馈 + AI对话记录 + 行为数据
- **规则引擎**：基于阈值的硬规则判断（如连续3天睡眠 < 5h）
- **AI增强**：AI综合分析多维数据，识别潜在风险模式

#### A6：趋势分析引擎

- **追踪维度**：睡眠质量、情绪趋势、运动依从性、症状频次
- **输出**：量化改善指标（如"睡眠质量提升40%"）
- **应用**：患者端激励展示 + 医生端决策参考

## 四、系统化管理方案

### 4.1 分层管理体系

系统根据体质评估结果自动分配管理等级，实现"分层管理、精准施策"：

| 等级 | 适用人群 | 评估频率 | 随访频率 | 日记要求 | 复诊引导 | 管理目标 |
|------|---------|---------|---------|---------|---------|---------|
| 🟢 低风险 | 平和质/低风险 | 每6个月 | 每月1次自动 | 鼓励但不强制 | 按季节提醒 | 维持健康，预防偏颇 |
| 🟡 中风险 | 偏颇体质/亚健康 | 每3个月 | 每2周1次 | 建议每日，AI提醒 | AI分析驱动 | 纠偏调理，改善体质 |
| 🔴 高风险 | 多兼夹/慢病并发 | 每月1次 | 每周1次 | 必须每日，异常报警 | 主动约诊+医生通知 | 密切监控，防止恶化 |
| ⚫ 危急 | 急性风险信号 | 即时 | 即时 | 即时 | 立即就医引导 | 紧急干预 |

**等级动态调整**：随访反馈改善则降级，出现风险信号则升级。

## 五、数据模型

### 5.1 核心数据实体

```
┌──────────┐     ┌──────────────────┐     ┌──────────────┐
│   User   │──→  │ PatientProfile   │──→  │ FollowupPlan │
│ (用户)    │     │ (患者档案)        │     │ (随访计划)    │
└──────────┘     └──────────────────┘     └──────────────┘
     │                   │                       │
     ├→ ConstitutionAssessment (体质评估)          ├→ FollowupEnrollment (随访注册)
     │       └→ HealthPlan (养生方案)              │       └→ FollowupRecord (随访记录)
     ├→ DiaryEntry (健康日记)                      │
     ├→ ChatSession → ChatMessage (AI对话)         │
     └→ DoctorProfile (医生档案) ─────────────────┘
```

### 5.2 数据模型详情

| 模型 | 字段 | 说明 |
|------|------|------|
| **User** | phone, openid, name, avatar, role | 基础用户，支持手机号+微信登录 |
| **PatientProfile** | gender, birthDate, height, weight, allergies, medicalHistory, riskLevel, primaryConstitution | 患者健康档案 |
| **DoctorProfile** | title, department, specialty, licenseNo | 医生职业信息 |
| **ConstitutionAssessment** | answers(JSON), scores(JSON), primaryType, secondaryTypes, aiAnalysis | 体质评估完整记录 |
| **HealthPlan** | dietAdvice, exerciseAdvice, lifestyleAdvice, emotionAdvice, acupointAdvice | 五维养生方案(JSON) |
| **DiaryEntry** | date, sleepHours, sleepQuality, moodScore, exerciseMinutes, exerciseType, dietNote, symptoms | 每日健康日记 |
| **ChatSession / ChatMessage** | messages with role, metadata(intent/safety) | AI对话记录 |
| **FollowupPlan** | targetConstitutions, targetRiskLevels, intervalDays, startDate, endDate | 随访计划 |
| **FollowupEnrollment** | plan↔patient 关联 | 随访注册 |
| **FollowupRecord** | scheduledDate, completedDate, status, feedback(JSON), aiSummary, riskFlag | 随访记录 |
| **ReminderLog** | type, note | 提醒记录 |
| **Hospital** | name, level, address, config(JSON功能开关) | 医院/机构 |

### 5.3 关键枚举

| 枚举 | 值 |
|------|---|
| **Role** | PATIENT, DOCTOR, ADMIN |
| **ConstitutionType** | BALANCED, QI_DEFICIENCY, YANG_DEFICIENCY, YIN_DEFICIENCY, PHLEGM_DAMPNESS, DAMP_HEAT, BLOOD_STASIS, QI_STAGNATION, SPECIAL |
| **RiskLevel** | LOW, MEDIUM, HIGH, CRITICAL |
| **FollowupStatus** | PENDING, COMPLETED, MISSED, CANCELLED |

## 八、安全与合规

### 8.1 AI 能力边界

| 能力 | 状态 | 说明 |
|------|------|------|
| 养生建议 | ✅ 支持 | 饮食、运动、起居、情志、穴位 |
| 疾病诊断 | ❌ 禁止 | 明确告知用户需就医 |
| 处方开具 | ❌ 禁止 | 不替代医生专业诊疗 |
| 紧急处置 | ⚠️ 拦截 | 立即引导就医，不做处置建议 |

### 8.2 紧急拦截机制

- **30+ 紧急关键词**硬编码拦截（胸痛、中风、大出血、"不想活"等）
- **规则引擎优先级高于AI模型输出**
- 拦截触发 → 立即引导拨打120或专业热线
- 拦截记录上报医生端形成闭环

### 8.3 数据安全

- JWT认证，7天有效期，httpOnly Cookie存储
- 医疗数据加密存储
- 角色权限控制（患者/医生/管理员）
- API接口级别的权限验证
